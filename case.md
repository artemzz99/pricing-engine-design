# Case: Personalized Pricing & Offer Engine for Retail Banking

---

## 1. Контекст

Крупный розничный банк (~5 млн клиентов) обслуживает массовый сегмент физлиц. У банка есть портфель продуктов (карты, депозиты, кредиты, премиальные пакеты), каждый из которых имеет набор настраиваемых параметров — **тариф**. Сегодня тарифы стандартизированы: клиенты видят одинаковые ставки, одинаковые условия кэшбэка и одинаковые промо-акции.

Работа с существующей клиентской базой — расширение продуктовой корзины, удержание активных клиентов, персонализация условий — ведётся по правилам, заданным вручную: маркетинг определяет сегменты и присваивает им пресеты офферов. Цикл обновления правил — недели. Управление коммуникациями (push-уведомления, sms/email) и продвижение продуктов в online-каналах (баннеры, кнопки и пр.) строится на основе  систем, оптимизирующих Propensity-to-buy и Propensity-to-click метрики соответственно.

---

## 2. Продуктовый каталог

Рассматриваются **четыре продукта**, которые формируют бóльшую часть прибыли розничного блока.

### 2.1 Дебетовая карта

Платёжный инструмент с возможностью получать кэшбэк за покупки.

**Настраиваемые параметры (тариф):**
- Базовая ставка кэшбэка (%)
- Повышенная ставка в выбранных категориях (%)
- Стоимость обслуживания (₽/мес) — может быть обнулена при выполнении условий
- Лимиты снятия наличных

**Доходы банка:**
- Interchange — комиссия от платёжной системы при безналичных покупках клиента

**Расходы банка:**
- Кэшбэк — основная статья расходов

**Риски:** нет кредитного риска (клиент тратит свои средства).

### 2.2 Кредитная карта

Возобновляемая кредитная линия с возможностью льготного периода.

**Настраиваемые параметры (тариф):**
- Процентная ставка (% годовых)
- Кредитный лимит (₽)
- Длительность льготного периода (дни)
- Кэшбэк за покупки (%)
- Стоимость обслуживания (₽/мес)
- Комиссия за снятие наличных

**Доходы банка:**
- Проценты по кредиту (клиент платит за использование средств вне льготного периода)
- Interchange

**Расходы банка:**
- Кэшбэк
- Стоимость фондирования (привлечение средств для выдачи кредитов)
- **Risk Cost** — ожидаемые потери из-за дефолтов (невозвратов)

**Риски:** есть **кредитный риск** — клиент может не вернуть задолженность. Требуется учёт Risk Cost при расчёте доходности.

### 2.3 Депозит / Накопительный счёт

Сберегательный инструмент с начислением процентов на остаток.

**Настраиваемые параметры (тариф):**
- Процентная ставка (% годовых)
- Срок (для депозита — фиксированный; для накопительного — нет)
- Условия пополнения / снятия
- Капитализация (ежедневная / ежемесячная / в конце срока)
- Порог суммы для повышенной ставки

**Доходы банка:**
- Маржа между ставкой привлечения (депозит) и ставкой размещения (кредиты / инвестиции)

**Расходы банка:**
- Проценты по депозиту — выплачиваются клиенту

**Риски:** нет кредитного риска, но есть **ликвидностный** — клиент может забрать средства раньше срока (для накопительного счёта — в любой момент).

### 2.4 Премиальный пакет

Пакет привилегий, объединяющий бенефиты из нескольких продуктовых категорий: бесплатные проходки в бизнес-залы аэропортов, страховка в путешествиях, карта Mir Supreme.

**Условия активации (тариф):**
- Бесплатно при остатках на счетах/депозитах ≥ 1,5 млн ₽
- Бесплатно при POS-оборотах ≥ 150 тыс. ₽/мес
- При невыполнении обоих условий — 2 000 ₽/мес

**Доходы банка:**
- Подписка 2 000 ₽/мес (при отсутствии условий)
- **Основная ценность — структурная:** стимулирует клиента наращивать остатки и обороты → доход от депозитов и interchange по другим продуктам

**Расходы банка:**
- Себестоимость привилегий: lounge access, страховка, выпуск и обслуживание Mir Supreme

**Риски:** минимальные; основной риск — что привилегии не окупаются структурным эффектом на портфель.

## 3. Кросс-продуктовые эффекты

Ключевая особенность задачи: **изменение тарифа по одному продукту влияет на доходность других продуктов.** Это делает задачу существенно сложнее, чем набор независимых решений по каждому продукту.

| Действие | Прямой эффект | Структурный эффект на другие продукты |
|---|---|---|
| Повышенный кэшбэк на дебетовой карте | ↑ расходы на кэшбэк (прямой убыток) | ↑ POS-обороты → ↑ interchange дохода · ↑ привычка использовать карту → ↑ вероятность хранить остаток на счёте → ↑ доход от размещения |
| Активация Премиального пакета | Расходы на привилегии | ↑ остатки ≥ 1,5 млн (чтобы не платить 2к/мес) → ↑ доход от депозитов · ↑ обороты ≥ 150к → ↑ interchange · ↑ NPS → ↓ churn |
| Повышенная ставка по депозиту | ↑ расходы на проценты | ↑ остатки → выполнение условий Премиального пакета → доступ к привилегиям → ↑ обороты |
| Снижение ставки по кредитной карте | ↓ доход от процентов | ↑ активация КК → ↑ POS-обороты → ↑ interchange · ↑ кросс-продажи (если клиент доволен условиями КК, он рассматривает депозит в том же банке) |

---

## 4. Текущие проблемы

1. **Неоптимальное распределение бюджета.** Ставки и промо для всех одинаковые. Клиент с высоким потенциалом получает тот же оффер, что и клиент с низким, — и либо недополучает ценности (уходит), либо получает избыточную (деньги потеряны).
3. **Short-term bias.** Оптимизация под быструю конверсию без учёта поведения клиента в горизонте года. Клиент, который принёс бы высокий доход через 6–9 месяцев, может отсекаться на моменте подбора оффера из-за низкого Propensity-to-buy и Propensity-to-click продукта.
4. **Игнорирование кросс-эффектов.** Решения по тарифам принимаются по каждому продукту отдельно. Структурное влияние одного продукта на другие (см. раздел 3) не учитывается — что приводит к субоптимальным комбинациям.

## 5. Целевая метрика: Доход с клиента

**Доход с клиента за период T** — суммарный Gross Income (GI) по всем продуктам клиента за горизонт T, за вычетом прямых расходов, связанных с обслуживанием этих продуктов.

Для каждого продукта *i* клиента *c*:

**GI(c, i) = Выручка(c, i) − Прямые расходы(c, i)**

где:
- **Выручка** — interchange, проценты по кредиту, маржа по депозиту, подписка (Премиум)
- **Прямые расходы** — кэшбэк, проценты по депозиту, себестоимость привилегий, Risk Cost (для кредитных продуктов)

**Доход с клиента:**

**GI(c) = Σ GI(c, i) по всем продуктам i**

Горизонт T = **12 месяцев** — компромисс между горизонтом планирования и возможностью измерить результат по факту.

## 6. Задача

Спроектировать систему, которая для каждого клиента выбирает **комбинацию тарифов по всем доступным ему продуктам**, максимизирующую **доход банка с этого клиента** (GI) в горизонте 12 месяцев, при соблюдении бюджетных и продуктовых ограничений.

## Глоссарий

- **Тариф (tariff):** набор параметров продукта, конфигурируемых для клиента (ставка, кэшбэк, лимит, срок, комиссии и т.п.).
- **Пресет:** фиксированная комбинация значений параметров тарифа, согласованная продуктовой командой.
- **Interchange:** комиссия, которую банк-эмитент получает от платёжной системы при безналичной покупке клиента.
- **Risk Cost:** ожидаемые потери по кредитному продукту из-за defaults (невозвратов).
- **POS-обороты:** объём безналичных покупок клиента за период.
- **Time-to-offer:** время от формирования оффера до его доставки клиенту.
- **GI (Gross Income):** доход по продукту = выручка − прямые расходы (кэшбэк, проценты, Risk Cost).
- **NPAT (Net Profit After Tax):** чистая прибыль по продукту после налогов.
